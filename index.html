Build from Zero

A complete guide to setting up, building, and deploying the Infinity application with authentication.

01 — Overview

What You Are Building

A Next.js application with Supabase authentication, Google OAuth, Stripe payments, and protected routes — deployed on Vercel behind Cloudflare.

How this guide works

We build in three layers: UI first (see something on screen), then backend (data, auth, payments), then deploy (live on the internet). This way you always have something working.

The Flow

Install tools→ Create Next.js app→ Build UI pages→ Connect backend→ Deploy

Architecture

User→ Cloudflare→ Vercel→ Supabase
02 — Foundation

Technology Stack

Every tool in the stack and why it exists.

Next.js

React framework. File-based routing, server components, API routes. App Router.

shadcn/ui

Copy-paste UI components on Radix + Tailwind. You own the code.

Supabase

PostgreSQL, auto-generated APIs, auth, row-level security.

Vercel

Hosting. Push to GitHub, auto-deploy. SSL and CDN included.

Cloudflare

DNS, CDN, WAF, DDoS protection. Traffic passes through first.

Stripe

Payments and subscriptions. Start in sandbox mode.

Google Cloud

OAuth 2.0 for "Sign in with Google".

GitHub

Private repo. Source control and deployment triggers.

Lucide

Beautiful, consistent icon set. Tree-shakeable React components.

04 — Build UI First

Create the Next.js App

No backend yet. Just get a working app on your screen.

Initialize the Project

Create a new folder and run:

npx create-next-app@latest my-app --typescript --tailwind --eslint --app --src-dir --import-alias "@/*" --use-npm

This gives you TypeScript, Tailwind, ESLint, and the App Router. Test it:

# Move into the project
cd my-app

# Start dev server
npm run dev

# Open http://localhost:3000
# You should see the Next.js welcome page

Install shadcn/ui

Follow the official setup: ui.shadcn.com/create

npx shadcn@latest init

When prompted: TypeScript, Default style, Slate color, CSS variables: Yes.

Add components as you need them:

npx shadcn@latest add button card input label

Each command copies component source into src/components/ui/. You own this code.

Install Lucide Icons

shadcn/ui uses Lucide for all icons. Install the package:

npm install lucide-react

Use icons in any component:

// Any component file
import { ArrowRight, Check, Copy, Mail } from 'lucide-react'

export default function Demo() {
  return (
    <div>
      <Mail className="h-4 w-4" />
      <ArrowRight className="h-5 w-5 text-gray-500" />
      <Check className="h-4 w-4 text-green-500" />
    </div>
  )
}

Browse all icons at lucide.dev. Import by name — only the icons you use are bundled.

Rules

  • Use Tailwind classes. No inline styles.
  • Use the cn() utility for conditional classes.
  • Stick to the grayscale palette. No random colors.
  • Spacing in multiples of 4px (4, 8, 12, 16, 20, 24, 32).
  • Use Lucide icons. Size with h-4 w-4 (default), h-5 w-5, etc.

Authentication

Wire up Supabase auth to the login page and protect routes.

Supabase Client

// src/lib/supabase.ts
import { createClient } from '@supabase/supabase-js'

export const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!
)

Middleware (protects routes)

// src/middleware.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse } from 'next/server'

export async function middleware(request) {
  let response = NextResponse.next({ request })

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    { cookies: { getAll() { return request.cookies.getAll() },
      setAll(c) { c.forEach(({ name, value }) => request.cookies.set(name, value))
        response = NextResponse.next({ request })
        c.forEach(({ name, value, options }) => response.cookies.set(name, value, options)) }}}
  )

  const { data: { user } } = await supabase.auth.getUser()

  if (!user && request.nextUrl.pathname.startsWith('/dashboard')) {
    return NextResponse.redirect(new URL('/login', request.url))
  }

  return response
}

export const config = { matcher: ['/((?!_next|favicon.ico|.*\\.(?:svg|png|jpg)$).*)'] }

Wire Up Login Page

// src/app/(auth)/login/page.tsx
'use client'
import { useState } from 'react'
import { useRouter } from 'next/navigation'
import { supabase } from '@/lib/supabase'
import { Button } from '@/components/ui/button'
import { Input } from '@/components/ui/input'
import { Card, CardContent, CardHeader, CardTitle } from '@/components/ui/card'

export default function LoginPage() {
  const [email, setEmail] = useState('')
  const [password, setPassword] = useState('')
  const router = useRouter()

  const handleLogin = async (e) => {
    e.preventDefault()
    const { error } = await supabase.auth.signInWithPassword({ email, password })
    if (!error) router.push('/dashboard')
  }

  const handleGoogle = async () => {
    await supabase.auth.signInWithOAuth({
      provider: 'google',
      options: { redirectTo: `${window.location.origin}/api/auth/callback/google` }
    })
  }

  return (
    <div className="flex min-h-screen items-center justify-center">
      <Card className="w-full max-w-sm">
        <CardHeader><CardTitle>Sign In</CardTitle></CardHeader>
        <CardContent>
          <form onSubmit={handleLogin} className="space-y-4">
            <Input type="email" placeholder="Email" onChange={e => setEmail(e.target.value)} />
            <Input type="password" placeholder="Password" onChange={e => setPassword(e.target.value)} />
            <Button type="submit" className="w-full">Sign In</Button>
            <Button type="button" variant="outline" className="w-full" onClick={handleGoogle}>Google</Button>
          </form>
        </CardContent>
      </Card>
    </div>
  )
}

21 — AI & Tools

Rules & Conventions

Add these to your context file. AI and humans follow the same rules.

Code Rules

## Code Rules
- TypeScript strict mode. No any types.
- Server Components by default.
- cn() for class merging. No inline styles.
- Components in src/components/. Utils in src/lib/.
- One component per file. Named exports for libs.
- Icons from lucide-react only. No other icon packages.

Git Rules

## Git Rules
- Messages: "type: description" (feat, fix, docs, refactor)
- Never commit .env.local or node_modules.
- One logical change per commit.
- Branch per feature: feat/login, fix/auth-redirect.

Security Rules

## Security Rules
- Never expose SUPABASE_SERVICE_ROLE_KEY to client.
- Never expose STRIPE_SECRET_KEY to client.
- All tables must have RLS policies.
- Validate input server-side.
- HTTPS only in production.

Debug an Issue

Login works in dev but 400 error in production:
"invalid_grant: code challenge mismatch"

Debug step by step:
1. Check Supabase auth config
2. Verify env vars match between .env.local and Vercel
3. Check redirect URIs in GCP
4. Check middleware for issues
5. Provide the fix

Code Review

Review for:
1. Security (SQL injection, XSS, exposed secrets)
2. Performance (re-renders, missing indexes)
3. TypeScript errors or any types
4. Accessibility (missing labels, keyboard traps)
5. Design system adherence

Flag severity (critical/warning/info). Provide fixes.
